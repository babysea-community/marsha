# Font Awesome Configuration

## Integration

- **Method:** SVG+JS Kit
- **Kit ID:** `1b8aa472ce`
- **Embed:** `https://kit.fontawesome.com/1b8aa472ce.js`
- **Compatibility with Older Versions:** disabled

## Defaults

- **Style:** solid
- **Family:** classic
- **Brands:** use `fa-brands` when a Font Awesome brand icon exists
- **Custom Kit Icons:** use `fa-kit` only for icons uploaded to this Kit

## Import Pattern

Use `FontAwesomeIcon` or `createFontAwesomeIcon` from `components/icons/font-awesome-icon.tsx`. App UI icons render as Kit-backed `<i>` tags so the hosted Kit can replace them with SVGs.

Do not add custom classes that start with `fa-` to Font Awesome icon elements. With SVG+JS and compatibility disabled, Font Awesome treats `fa-*` classes as family, utility, or icon-name tokens; unrelated `fa-*` markers can resolve as missing icons and render as question marks.

## Conventions

- General UI glyphs use solid Font Awesome icons.
- Company/provider identity icons use Font Awesome brands when available, with conservative solid fallbacks for providers that do not have public Font Awesome brand marks.
- The Font Awesome Kit script is loaded once in `app/layout.tsx` with `lazyOnload`, `data-auto-replace-svg="nest"`, and CSP allowlist support in `lib/security/csp.ts`. SVG+JS mutates icon markup, so loading it after page load and nesting generated SVGs inside the original `<i>` reduces React hydration/reconciliation risk on server-rendered pages.
